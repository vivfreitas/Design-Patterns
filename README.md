# ☕ PADRÕES DE DESIGN
___
Os padrões de design **(Design Patterns)** são soluções que você pode usar para resolver problemas frequentes que ocorrem durante o ciclo de vida do desenvolvimento.
**Eles não são** uma biblioteca/framework que você instala a dependência ou um bloco de código para serem usados, mas sim um guia de como resolver problemas de estruturas.
### Ao aplica-los você terá uma melhora:

- Padronização.
- Manutenibilidade.
- Eficiência.
- Baixo acoplamento.
- Escalabilidade de projeto.
___
## 🦝 Categorias de padrões

- **Padrões Criacionais**: Fornecem mecanismos de criação de objetos que aumentam a flexibilidade e a reutilização de código.
- **Padrões Estruturais**: Explicam como montar objetos e classes em estruturas maiores, enquanto ainda mantém as estruturas flexíveis e eficientes.
- **Padrões Comportamentais**:  cuidam da comunicação eficiente e da assinalação de responsabilidades entre objetos.
